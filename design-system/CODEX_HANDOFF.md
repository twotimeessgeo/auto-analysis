# TWOTIMESS Design System Handoff

다른 Codex에게 아래 지침을 그대로 전달하면 됩니다.

## 전달용 메시지

첨부한 `design-system/` 폴더를 이 프로젝트의 단일 UI 기준으로 사용해주세요.
시험형 그래프나 교육용 비교 화면이 포함된다면 `GRAPH_AND_EXAM_GUIDE.md`까지 같이 기준으로 삼아주세요.
인문지리 주제라면 `HUMAN_GEOGRAPHY_GUIDE.md`를 우선 참고해서 자연지리형 카드 구조를 그대로 복제하지 말고, 같은 시각 언어 안에서 인문지리형 정보 구조로 바꿔주세요.

반드시 지켜야 할 사항:

1. 전체 톤은 `흑백 + 종이색 배경`입니다.
2. `Pretendard` 계열 폰트를 사용합니다.
3. 모서리는 둥글리지 말고 `직각`으로 유지합니다.
4. 그라데이션, 유리 효과, box-shadow, 과한 색상 포인트를 넣지 않습니다.
5. 제목은 강하고 크게, 본문은 얇고 차분하게 구성합니다.
6. 패널, 표, 차트는 모두 같은 보더 체계를 사용합니다.
7. 차트는 검정/회색 중심으로, 정보 전달이 우선입니다.
8. 모바일에서도 같은 위계를 유지해야 합니다.

작업 방식:

1. 먼저 `tokens.css`, `base.css`, `components.css`, `patterns.css`를 기준으로 공통 구조를 잡습니다.
2. 기존 화면을 임의로 새로 디자인하지 말고, 공통 패턴 안으로 재배치합니다.
3. 새 컴포넌트가 필요하면 기존 컴포넌트의 변형으로 해결하고, 완전히 다른 언어를 만들지 않습니다.
4. 결과물에는 다음이 포함되어야 합니다.
   - 적용한 레이아웃 구조 설명
   - 어떤 공통 컴포넌트를 썼는지
   - 모바일 대응 방식
   - 토큰에서 바꾼 값이 있다면 그 이유
   - 그래프와 통계가 `GRAPH_AND_EXAM_GUIDE.md` 원칙을 어떻게 반영했는지
   - 인문지리라면 `HUMAN_GEOGRAPHY_GUIDE.md` 기준으로 어떤 그래프 유형과 비교 구조를 골랐는지

금지:

- 둥근 pill 버튼
- 보라색 계열 기본 강조색
- 유리질 반투명 카드
- 과한 애니메이션
- 기본 system font로 회귀
- 패널마다 다른 보더 굵기 사용

승인 기준:

- 기존 TWOTIMESS 프로젝트와 같은 계열로 즉시 느껴져야 합니다.
- 정보를 많이 담아도 답답하지 않아야 합니다.
- 표, 그래프, 버튼, 필터가 한 화면에서 섞여도 위계가 무너지지 않아야 합니다.

## 짧은 프롬프트 버전

첨부한 `design-system/` 폴더를 기준으로 이 프로젝트의 UI를 재구성해주세요.
Pretendard, 흑백, 종이색 배경, 직각 패널, 1px 보더, 큰 제목, 차분한 본문, 무그라데이션, 무그림자 원칙을 지키고,
새 스타일을 임의로 만들기보다 기존 컴포넌트와 패턴을 재사용해 일관성을 맞춰주세요.
